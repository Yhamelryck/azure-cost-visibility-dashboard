# Azure Cost Visibility Dashboard

This project demonstrates how Azure Cost Management, Azure Monitor, Logic Apps, and Azure Workbooks can be used to improve visibility into Azure spending and alert stakeholders before costs become a problem.

## Project goal
Small organizations often struggle to understand where Azure costs come from. The goal of this project is to build a practical cost visibility solution that tracks spending, highlights trends, and notifies stakeholders when predefined thresholds are reached.

## What I want to build
- Cost visibility by service, resource group, and time period
- Budget or cost threshold alerts
- Email notifications through an automated workflow
- A dashboard for easier cost analysis
- A weekly comparison report showing cost changes over time

## Azure services used
- Azure Cost Management
- Azure Monitor
- Azure Workbooks
- Logic Apps
- Azure Function

## Planned workflow
1. Track Azure spending using Azure Cost Management
2. Group and interpret costs in a business-readable way
3. Create alerts for selected spending thresholds
4. Trigger a Logic App to send notifications
5. Visualize spending in a dashboard
6. Compare current and previous cost data in a weekly report

## Why this project matters
This project focuses on a real business problem: unexpected cloud costs. It demonstrates not only Azure knowledge, but also operational thinking, cost awareness, automation, and the ability to communicate technical data in a way that supports decision-making.
